내 프로젝트가 잘 되고 있는지,
웹에서 바로 확인하세요

지금은 프로젝트가 깔린 컴퓨터 앞에 앉아야만 진행 상황을 볼 수 있습니다. 이걸 로그인하는 안전한 웹 사이트에서 누구나 쉽게 보도록 만드는 계획입니다.

?지금 무엇이 불편한가요

우리 작업 방식은 프로젝트마다 "현재 무엇을 하고 있는지"와 "지금까지 나온 결과물"을 파일로 차곡차곡 남깁니다. 문제는 —

🖥️
이 내용을 보려면 그 프로젝트가 설치된 컴퓨터 앞에 직접 앉아야 합니다. 멀리 있는 고객은 "내 프로젝트가 지금 어디까지 됐지?"를 물어볼 수밖에 없습니다.

✓이렇게 바꿉니다

고객이 웹 사이트에 로그인하면, 자기 프로젝트가 지금 진행 중인지와 지금까지 나온 결과물(문서)을 보기 좋은 화면으로 바로 확인할 수 있게 합니다.

1

프로젝트가 스스로 현황을 보고합니다

작업이 한 단계 끝날 때마다 "지금 몇 번째 단계, 무슨 일을 하는 중"이라는 요약과 새로 나온 결과물 문서를 자동으로 올려보냅니다. 사람이 손으로 정리할 필요가 없습니다.

▼
2

안전한 비공개 보관함에 모입니다

올라온 내용은 외부에 공개되지 않는 우리만의 보관함(데이터베이스)에 저장됩니다. 고객별로 칸이 나뉘어 있어 서로의 내용은 절대 섞이지 않습니다.

▼
3

고객이 로그인해서 확인합니다

고객은 자기 계정으로 로그인해 자기 프로젝트만 봅니다. 팀(우리)은 전체를 한눈에 봅니다.

🔒
기밀이 최우선입니다. 고객의 진행 내용은 검색되거나 외부에 노출되지 않습니다. 오직 권한이 있는 사람이 로그인했을 때, 그 사람에게 허락된 프로젝트만 보입니다.

  • 지금 무슨 작업을 하는 중인지

⚙어떤 프로젝트가 대시보드에 보이나요

우리는 여러 프로젝트를 진행합니다. 그중 고객에게 보여줄 프로젝트만 골라서 대시보드에 연결합니다. 나 혼자 하는 토이·실험 프로젝트는 아무것도 올라가지 않고 대시보드에도 뜨지 않습니다.

연결한 프로젝트

대시보드에 노출 ✓

  • 어드민에서 발급한 정보로 연결 설정 파일을 직접 추가한 경우만
  • 현황·결과물이 안전한 보관함에 올라감
  • 고객이 로그인해 확인 가능
토이 / 개인 프로젝트

완전 비공개 🔒

  • 기본값은 "연결 안 함" — 따로 안 켜면 그대로
  • 데이터가 보관함에 들어가지 않음
  • 대시보드에 아예 나타나지 않음
🛡️
이중 안전장치. 프로젝트 쪽에서 연결을 켜지 않으면 아무것도 보내지 않고, 설령 실수가 있어도 서버가 미리 등록한 프로젝트만 받아들입니다. 잘못 노출될 일이 없습니다.

→어떻게 만들어 가나요

한 번에 다 만들지 않고, 작게 검증하며 단계적으로 넓혀갑니다.

  1. 현황을 또렷한 데이터로 만들기
    프로젝트가 "지금 무슨 단계인지"를 정확한 형식으로 적도록 정비합니다. 모든 화면의 기초입니다.
  2. 팀 전용으로 처음부터 끝까지 한 번 돌려보기
    프로젝트 1개로 "보고 → 저장 → 화면 표시"가 실제로 되는지 확인합니다.
  3. 고객 로그인 · 칸막이(기밀) 붙이기
    고객별 계정과 "내 것만 보이는" 안전장치를 추가합니다.
  4. 안정화
    접속 권한 관리, 최신 여부 표시, 결과물 버전 관리 등을 다듬습니다.
팀을 위한 기술 메모 (비개발자는 건너뛰셔도 됩니다)

· 현황 데이터 계약: docs/.orch-state.json에 빠진 project_id · current_step · status · updated_at 필드 추가 (가장 큰 병목, 최우선).

· 수집: git 공개 경유 대신, 프로젝트가 사이클 후 상태+산출물 md를 비공개 ingest API로 토큰 인증 push.

· 저장: 비공개 DB 3 테이블 — projects(최신 status) / artifacts(md+변환 html) / access(고객↔프로젝트 매핑).

· 어드민: 별도 서비스. 권장 스택 Next.js + Supabase(Auth + Postgres + RLS로 테넌트 격리). md→HTML은 기존 md2html 패턴 재사용.

· 옵트인(기본 off, 파일 직접 생성으로 확정): 프로젝트에 .claude/dashboard.json(enabled/project_id/project_name/ingest_url/token, .gitignore에 추가)을 직접 만들었을 때만 reporter 동작. 별도 커맨드·설치 플래그 없음. 토이 프로젝트는 파일 없음 → push no-op. 서버는 등록된 project_id + 유효 토큰만 수락(이중 게이트). 끄기는 파일 삭제 또는 enabled:false.

· r2_wf 템플릿은 무의존성 유지 — 대시보드는 템플릿 밖 별도 서비스이고, 템플릿엔 "상태 emit + (옵트인 시) push"만 더한다.
