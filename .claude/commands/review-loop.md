# review-loop

코드 리뷰 → 수정 사이클을 자동으로 N회 반복한다.

## Usage

```
/review-loop [횟수]
```

- `횟수` 생략 시 기본값 **5회**
- 예: `/review-loop 3`

## Instructions

`$ARGUMENTS`에서 반복 횟수를 읽는다. 값이 없으면 5를 사용한다.

아래 사이클을 **최대 N회** 반복한다. 단, 리뷰에서 지적 사항이 없으면 그 즉시 루프를 종료한다.

### 각 회차 절차

1. **code-reviewer 에이전트 실행**
   - 현재 브랜치에서 변경된 파일 전체를 대상으로 한다.
   - 에이전트에 전달할 컨텍스트:
     - 프로젝트 규칙: `any` 타입 금지, `console.log` 금지, `src/lib/`에 React/DOM import 금지, 미사용 import 금지
     - 네이밍 컨벤션: 컴포넌트 PascalCase, 일반 파일 camelCase, 타입 PascalCase
     - 타입스크립트 엄격 모드 준수
   - 에이전트 결과에서 **지적 사항 목록**을 추출한다.

2. **지적 사항 분류**
   - 지적 사항이 없으면 → "리뷰 통과: 지적 사항 없음"을 출력하고 루프 종료.
   - 지적 사항이 있으면 → 심각도(blocking / warning / suggestion) 순으로 정렬한다.

3. **수정 적용**
   - blocking 및 warning 항목을 모두 수정한다.
   - suggestion은 판단 후 적용 여부를 결정한다. 적용하지 않을 경우 이유를 한 줄로 기록한다.
   - 수정 완료 후 `npx tsc --noEmit`을 실행해 타입 에러가 없는지 확인한다.
   - 타입 에러가 남아 있으면 에러를 먼저 해소한 뒤 다음 회차로 넘어간다.

4. **회차 요약 출력**
   ```
   [회차 M/N] 지적 M건 (blocking X, warning Y, suggestion Z) → 수정 완료
   ```

### 루프 종료 후 최종 요약

모든 회차가 끝나면 아래 형식으로 출력한다:

```
=== review-loop 완료 ===
총 N회 실행 / 조기 종료: [예/아니오]
회차별 지적 건수: [1회차: M건, 2회차: K건, ...]
최종 상태: 타입체크 통과 여부
남은 미적용 suggestion 목록 (있을 경우)
```
